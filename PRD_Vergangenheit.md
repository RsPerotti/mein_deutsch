# PRD: Vergangenheit — Perfekt & Präteritum

**Project:** Mein Deutsch — Android PWA  
**Feature:** Past Tense Layer (Perfekt + Präteritum) in Verbs Module  
**Status:** Awaiting sign-off — all questions resolved  
**Date:** 2026-06-12  

---

## Problem Being Solved

Learners can currently only express themselves in the present tense. Without Perfekt and Präteritum, they cannot discuss anything in the past — the single biggest practical gap in the app. Perfekt is the dominant conversational past tense; Präteritum is essential in writing and for common verbs. Both need to be present for A1–B2 coverage to be meaningful.

---

## Success Criteria

1. All 91 root verbs have Präteritum conjugations (6 forms each), verified for accuracy.
2. Prefix variants have Perfekt and Präteritum data added.
3. The Verbs module presents a **Präsens / Vergangenheit** toggle — learners can drill each tense independently.
4. Three new exercise types function correctly and score objectively (no free-translation exercises):
   - Partizip II production
   - Auxiliary identification (haben vs. sein)
   - Full 6-person conjugation table (separate exercises for Präsens, Perfekt, Präteritum)
5. Each verb exercise displays the verb's **Regular / Irregular** label and **case requirement** (Akkusativ, Dativ, Nominativ, Genitiv) in close proximity.
6. No regressions in existing Präsens exercises or other modules.

---

## Scope

### In

- **Verbs module only** — no other modules touched.
- **Data: Präteritum conjugations** — add `prateritum` object (6 forms) to all 91 root verbs. Claude generates; Claude verifies.
- **Data: Prefix variants** — add Perfekt and Präteritum data to prefix verb entries. Root verbs completed first; variants second.
- **Data: Regular/Irregular mapping** — existing `type` field (`weak` → Regular; `strong`, `irregular`, `modal`, `mixed` → Irregular) surfaced as a display label. No new field needed.
- **Data: Case display** — existing `grammar.case_requirements` already present; surfaced visually near the Regular/Irregular label.
- **UI: Präsens / Vergangenheit tab/toggle** within the Verbs module. Extensible — Futur, Konjunktiv, etc. added as future tabs without rearchitecting.
- **Exercises (Vergangenheit tab):**
  - `partizip_ii` — given the infinitive, type the Partizip II form
  - `auxiliary_choice` — given the infinitive, select *hat* or *ist* (objective multiple choice)
  - `conjugation_table` — given a verb and tense, fill in all 6 person forms (separate exercises for Präsens, Perfekt, Präteritum)
- **Exercise engine** — `js/exercises.js` updated to handle the three new exercise types.

### Out

- Lesen & Hören module — untouched.
- Nouns, Adjectives, Adverbs, Prepositions — untouched.
- Translation exercises — excluded by design (objective checking not possible without AI).
- Futur, Konjunktiv, Passiv — noted as future tabs; not built now.
- Any backend, sync, or service worker changes.

---

## Constraints

- All answers must be objectively checkable (exact string match or predefined options). No AI-graded responses.
- Präteritum data generated by Claude and verified by Claude against authoritative conjugation references before any exercise is built on top of it.
- The Präsens / Vergangenheit toggle must not break the existing exercise stream for learners who haven't touched the new content yet.
- Build order: root verbs → prefix variants → exercise types → UI toggle.

---

## Proposed Approach

### Phase 1 — Data
1. Add `prateritum` conjugation object to all 91 root verbs in `js/data.js`.
2. Verify all Präteritum forms for accuracy.
3. Add Perfekt + Präteritum data to prefix verb entries.
4. Confirm `type` → Regular/Irregular mapping is consistent across all 91 entries.

### Phase 2 — Exercise Engine
5. Add three new exercise type handlers to `js/exercises.js`:
   - `partizip_ii`
   - `auxiliary_choice`
   - `conjugation_table` (tense-parameterised)
6. Generate new exercises for all verbs covering the new types.

### Phase 3 — UI
7. Add Präsens / Vergangenheit tab/toggle to the Verbs module view.
8. Wire exercise routing so each tab serves the appropriate exercise pool.
9. Add Regular/Irregular label + case badge to verb exercise cards.

### Phase 4 — Verification
10. Smoke-test all new exercise types across a sample of verbs (regular, irregular, modal, sein/haben auxiliaries).
11. Confirm zero regressions in existing Präsens exercises and all other modules.
